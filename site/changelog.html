● Changelog

Every release, since v0.1.

From a git-native decision graph to deterministic Architectural Conformance — the whole arc, newest first.

Jun 25, 2026
v0.31Google Antigravity support — one .hunch/ graph now serves Google’s agent IDE too. Grounding is automatic (Antigravity reads the project-root AGENTS.md Hunch already maintains); hunch init registers the MCP server in Antigravity’s global config — idempotent-merge and detection-gated, so it only writes when Antigravity is installed.
v0.30Intent-conformance — the inversion. Other guards ask “did this diff touch a guarded file?”; hunch conform asks whether the code still satisfies a decision's recorded intent, so silent drift is caught with no diff in scope. A new kind of correctness beside types and tests.
v0.29Inline intent capture — mark intent where it lives in code: hunch-why: in a comment becomes a Decision, hunch-rule: a file-scoped Constraint. A third capture source beside commit synthesis and corrections. Deterministic, idempotent.
v0.28hunch compare — multi-candidate verdict — rank N candidate branches/commits by architectural fit; the one that trips the fewest invariants, reverses no decisions, and adds the least sprawl wins. The “evaluate 5 solutions” check, deterministic. CLI hunch compare + MCP hunch_compare.
v0.25–0.27Runbook memory — the how layer. hunch runbook distills a reusable procedure from a commit range, and hunch_runbook(task) returns the proven steps for a recurring task. Plus self-healing grounding that keeps every assistant's docs fresh without a re-init.
v0.24Memory drift checks — hunch doctor now flags when memory falls out of sync with the code: a decision referencing a deleted file, a dangling supersede, a doc still marked “proposed” that references shipped code. Deterministic, advisory — a smoke detector for stale memory.
v0.23Retrieval eval harness — hunch eval scores retrieval (Recall@k, MRR) over a curated golden set and A/Bs the graph stream, so quality is a measured number, not a claim. No LLM in the scoring path — a stable CI signal.
v0.22Graph-augmented retrieval — uses your code's own connections to surface the cross-file evidence a “why” question needs and plain search misses. Measured +41.7pt recall@10.
Jun 23, 2026
v0.18Deep Synthesis — --deep gathers several independent takes on a change and reconciles them into one more-trustworthy note, trusting it more when they agree. Subscription-only, and it stays advisory until you confirm it.
v0.17Private memory overlay — keep sensitive decisions, bugs, and constraints in a separate private repo while your main repo stays public. Point HUNCH_PRIVATE_DIR at it and Hunch unions that store into every query and guard locally — MCP and the pre-edit hook see your private memory — but it’s never committed here, and the CI PR comment renders public-only by construction. hunch private sets the whole thing up in one command (a gitignored .hunch/local.json, no env var, no shell edit), with opt-in --auto-commit to push each captured decision to the repo it lands in.
v0.16Redundancy Guard — “this already exists” — coding agents work from a local context window, so they re-implement helpers that already live elsewhere. Hunch sees your whole codebase, so a diff that adds something already defined in another file is flagged with its existing location. Advisory — never blocks, tuned to stay quiet so a refactor that just moves code isn’t mistaken for a duplicate.
Jun 21, 2026
v0.15Veto — the Decision Guard — re-introducing an approach a decision rejected (the alternative that never existed in code) is now caught — blocking the commit, the CI guard, and the live edit with a cited receipt: what you rejected, what you chose instead. Only a reversal you’ve confirmed blocks, so day-one stays advisory. Also: relicensed Apache-2.0 and added CONTRIBUTING.
Jun 20, 2026
v0.14Causal Merge Verdict — hunch_merge_verdict replays a diff against memory and returns a cited BLOCK / WARN / PASS, showing the decision and the bug behind each rule — so a change that re-opens a closed bug is caught with its why. Also: supply-chain hardening and a Windows config heal.
v0.13Never Twice — corrections become enforced invariants — new hunch_record_correction MCP tool: when you correct the agent ("no — never call X here"), Hunch mints a scoped, human-confirmed Constraint that the pre-edit hook and CI guard then hold every assistant to. The UserPromptSubmit hook nudges the agent to persist a rule whenever a prompt reads like a correction.
Jun 18, 2026
v0.12CI Constraint Guard — hunch ci scaffolds a GitHub Action that runs hunch check on every PR, comments the affected invariants/decisions (con_/dec_ ids), and fails on a blocking one. Memory that blocks a merge, not just advises. Also: the guard fails loudly on an unresolvable --base, and Hunch went OS-agnostic with path-separator normalization across Windows/macOS/Linux.
v0.11On npm + safe strict mode — install with npm i -g @davesheffer/hunch. --strict fails a commit only on a direct, high-confidence, non-stale invariant — safe to enable on a shared repo. Also: parallel backfill and a Windows git-worktree hooks-dir fix.
