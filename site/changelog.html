● Changelog

Every release, since v0.1.

From a git-native decision graph to deterministic Architectural Conformance — the whole arc, newest first.

Jul 2, 2026
v1.01.0 — zero native SQLite dependency — the derived index now runs on Node’s built-in node:sqlite instead of better-sqlite3: no more install-time native builds, no node-gyp fallback failures, no locked-.node EPERM on Windows upgrades — the SQLite binding ships inside the Node binary itself. FTS5 ranking, WAL, and the recursive-CTE graph walks are unchanged. Requires Node ≥ 22.13 (the first LTS line with node:sqlite unflagged; Node 20 is past EOL).
Jul 2, 2026
v0.40One source of truth for memory — every mode, branch, worktree, teammate, agent — hunch shared --repo <url> is now unified: every capture (decisions, bugs, constraints, runbooks) routes through one resolver to one home, so nothing splits between the public and shared stores. The overlay anchors at the main worktree root, so a git worktree remove can no longer silently take the store down. A committed .hunch/team.json advertises the store, so a fresh clone auto-connects on hunch init — and any MCP-connected agent or CI job wires up the same way on server start. hunch doctor now reports the true mode and shouts a dead-pointer finding instead of silently reading empty memory.
v0.40Auto-commit ON by default, in every mode — a capture is now committed to whichever store it lands in without being asked: a private/shared overlay repo is committed and pushed; the public .hunch/ is committed only and rides your next push — Hunch never pushes or merges your code branch. Opt out with --no-auto-commit on init / private / shared.
v0.40Security — the team pointer is gated — .hunch/team.json is committed, so in a freshly cloned untrusted repo its URL is attacker-controlled and auto-consumed on MCP start. The clone now runs behind a strict URL allowlist (https / ssh / git / scp-style only) plus -- and a protocol kill-switch, so a smuggled git flag or ext:: transport can never turn opening a repo into code execution.
Jul 1, 2026
v0.39Decision-grounding — the doc≠graph spoke — the complement to Architectural Conformance (graph≠code): both keep memory that stays true, one holding the code to recorded intent, this one holding the docs to the graph. A decision gains an optional topic anchor, so current(topic) / history(topic) / rejected(topic) resolve the live decision from the superseded ones — backward-compatible, no schema bump, existing graphs load unchanged.
v0.39Read-time grounding + the anchor-stale drift — the pre-edit hook now surfaces a file's topic-anchored decisions with follow the graph, not a stale doc precedence, and what each one rejected. A new deterministic anchor-stale drift fires when a file is still anchored to a superseded decision while a current one exists for its topic — shown in hunch doctor and gate-able via hunch drift, which exits non-zero on anchor-stale drift or a topic collision. No semantic guessing — it only fires on explicit anchors.
v0.39One live decision per topic — by construction — hunch_record_decision is now gated by a store-scoped uniqueness guard: it refuses a second live decision for a topic, so you never silently end up with two. New MCP hunch_capture_decision returns a one-question-at-a-time grilling protocol plus a capture-session token, and hunch_current_decision(topic) reads the live one; un-token'd writes still work but get nudged toward /capture. And hunch reconcile-topics catches the one invariant a git merge can violate — >1 live decision per topic — and exits non-zero, so you wire it into a post-merge hook / CI.
v0.39hunch heal + /capture and /heal — read-only doc↔graph reconciliation, scaffolded by hunch init as slash commands. It surfaces where a doc and the graph disagree and walks the capture; it never rewrites your prose silently.
Jun 27, 2026
v0.38.3Overlay auto-commit safety — hardened the optional shared/private memory auto-sync so it can never touch your code repo: a memory sync is purely additive JSON, so the commit is refused if the staged set contains any deletion or non-JSON file, and the overlay is initialized as its own git repo. Recording/query were never affected. If you use --auto-commit, upgrade.
v0.38Architectural Conformance for AI code — the wedge a linter can't reach. hunch conform --add records a semantic architectural invariant — layering (“controllers must not reach the DB”), must-reach (“charge must verify the session”), dependency direction — as a deterministic graph-reachability check. The gate (hunch check --strict + the CI PR gate) then blocks any AI change that breaks it, with the receipt: the decision behind the rule and the bug it prevents reopening. No model in the gate — the architectural violations a pattern-based linter passes clean.
v0.37Enforcement at a glance + grounding everywhere — hunch status shows what's armed, what's waiting to confirm, and what went stale, in one screen. And a recorded rule now propagates to every assistant's grounding — Claude Code, Cursor, Copilot, Windsurf, AGENTS.md — not just CLAUDE.md, so a rule taught once is held by all of them.
v0.36Parsed-import rule matcher + auto-derive — --forbid-dep matches the parsed import, so a comment or string naming a module can't false-positive and a submodule is still caught — and it's immune to staleness, blocking the real violation across the file's whole life. Correcting “never import X” auto-derives that matcher, validated against your real dependencies so it never mints a rule that can't fire.
Jun 26, 2026
v0.35Content-matched constraints + symlink-safe hooks — give a blocking rule a content matcher and it blocks the actual change across the file's life, while staying quiet on edits that don't break it — instead of relaxing to advisory after the file is touched. Plus the edit-time hook now delivers under symlinked repo roots (/tmp, /var, symlinked $HOME).
v0.34One memory across branches & worktrees — two-way auto-sync keeps the .hunch/ graph consistent across every branch and git worktree with no manual push/pull. Edit memory anywhere in the repo; it's the same source of truth everywhere — completely seamless.
v0.32–0.33Seamless multi-branch / worktree memory — the graph resolves through the git common directory, so linked worktrees and concurrent branches share one memory automatically — no per-worktree setup.
Jun 25, 2026
v0.31Google Antigravity support — one .hunch/ graph now serves Google’s agent IDE too. Grounding is automatic (Antigravity reads the project-root AGENTS.md Hunch already maintains); hunch init registers the MCP server in Antigravity’s global config — idempotent-merge and detection-gated, so it only writes when Antigravity is installed.
v0.30Intent-conformance — the inversion. Other guards ask “did this diff touch a guarded file?”; hunch conform asks whether the code still satisfies a decision's recorded intent, so silent drift is caught with no diff in scope. A new kind of correctness beside types and tests.
v0.29Inline intent capture — mark intent where it lives in code: hunch-why: in a comment becomes a Decision, hunch-rule: a file-scoped Constraint. A third capture source beside commit synthesis and corrections. Deterministic, idempotent.
v0.28hunch compare — multi-candidate verdict — rank N candidate branches/commits by architectural fit; the one that trips the fewest invariants, reverses no decisions, and adds the least sprawl wins. The “evaluate 5 solutions” check, deterministic. CLI hunch compare + MCP hunch_compare.
v0.25–0.27Runbook memory — the how layer. hunch runbook distills a reusable procedure from a commit range, and hunch_runbook(task) returns the proven steps for a recurring task. Plus self-healing grounding that keeps every assistant's docs fresh without a re-init.
v0.24Memory drift checks — hunch doctor now flags when memory falls out of sync with the code: a decision referencing a deleted file, a dangling supersede, a doc still marked “proposed” that references shipped code. Deterministic, advisory — a smoke detector for stale memory.
v0.23Retrieval eval harness — hunch eval scores retrieval (Recall@k, MRR) over a curated golden set and A/Bs the graph stream, so quality is a measured number, not a claim. No LLM in the scoring path — a stable CI signal.
v0.22Graph-augmented retrieval — uses your code's own connections to surface the cross-file evidence a “why” question needs and plain search misses. Measured +41.7pt recall@10.
Jun 23, 2026
v0.18Deep Synthesis — --deep gathers several independent takes on a change and reconciles them into one more-trustworthy note, trusting it more when they agree. Subscription-only, and it stays advisory until you confirm it.
v0.17Private memory overlay — keep sensitive decisions, bugs, and constraints in a separate private repo while your main repo stays public. Point HUNCH_PRIVATE_DIR at it and Hunch unions that store into every query and guard locally — MCP and the pre-edit hook see your private memory — but it’s never committed here, and the CI PR comment renders public-only by construction. hunch private sets the whole thing up in one command (a gitignored .hunch/local.json, no env var, no shell edit), with opt-in --auto-commit to push each captured decision to the repo it lands in.
v0.16Redundancy Guard — “this already exists” — coding agents work from a local context window, so they re-implement helpers that already live elsewhere. Hunch sees your whole codebase, so a diff that adds something already defined in another file is flagged with its existing location. Advisory — never blocks, tuned to stay quiet so a refactor that just moves code isn’t mistaken for a duplicate.

Only a reversal you’ve confirmed blocks, so day-one stays advisory. Also: relicensed Apache-2.0 and added CONTRIBUTING.

The UserPromptSubmit hook nudges the agent to persist a rule whenever a prompt reads like a correction.
